# Resume Coverage And Apply-Safe Template Catalog

Resume generation will use a coverage policy based on career-family fit, gap coverage, and the user's resume tailoring style instead of silently hiding every weak-fit work-history record. Weak-fit and gap-coverage guidance is app-only review guidance, while exported resumes remain normal ATS-safe documents; new apply-flow templates may be added only when they provide functional layout variety and pass the same apply-safe renderer and benchmark bar before becoming approval-eligible.

**Considered Options**

- Hide weak-fit roles by default in all modes: rejected because it can create unexplained work-history gaps and omit useful dev-adjacent history.
- Include every role by default: rejected because unrelated history can weaken a focused software resume.
- Add visual-only templates before benchmarking: rejected because approval and apply flows must not expose unproven resume layouts.
